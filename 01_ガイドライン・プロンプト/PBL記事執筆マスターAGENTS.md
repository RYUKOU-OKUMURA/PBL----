# PBL記事執筆AGENTS（全シリーズ共通・大元仕様）

このドキュメントは、PBL（奥村整体院）のブログ記事を、誰が書いても同じ品質で再現できるようにする標準運用仕様（マスターAGENT）です。文体・構成・保存運用を変数化しており、シリーズの有無やテーマに依らず利用できます。

- 参照1（文体）: `{{STYLE_GUIDE_FILE}}`（初期値: HPブログ記事/agent/PBL記事スタイルガイド.md）
- 参照2（運用）: `{{OPS_GUIDE_FILE}}`（初期値: HPブログ記事/agent/ブログ記事執筆ガイドライン.md）
- 参照3（構成/シリーズ別）: `{{STRUCTURE_FILE}}`（任意。シリーズごとの構成案。初期値: HPブログ記事/むち打ち症リサーチ/むち打ち症シリーズブログ構成案.md）

---

## 0. 設定（プロジェクト共通／シリーズは任意）
- 必須
  - `STYLE_GUIDE_FILE`: 文体ガイド（奥村龍晃氏の文体定義書）
  - `OPS_GUIDE_FILE`: 運用ガイド（進捗管理・保存運用）
- 任意（シリーズ運用時に使用）
  - `SERIES_NAME`: シリーズ名（例: むち打ち症シリーズ）
  - `SERIES_DIR`: 記事の保存先ディレクトリ（例: HPブログ記事/むち打ち症リサーチ/）
  - `STRUCTURE_FILE`: 構成案ファイル（各回の固定タイトル・要点）

初期値（必要に応じて差し替え）
- `STYLE_GUIDE_FILE` = HPブログ記事/agent/PBL記事スタイルガイド.md
- `OPS_GUIDE_FILE` = HPブログ記事/agent/ブログ記事執筆ガイドライン.md
- `SERIES_DIR` = HPブログ記事/むち打ち症リサーチ/
- `STRUCTURE_FILE` = HPブログ記事/むち打ち症リサーチ/むち打ち症シリーズブログ構成案.md

---

## 1. 目的と適用範囲
- 目的: PBLの全ブログ記事で、文体・構成・保存運用を揃えて一貫性高く執筆する。
- 適用範囲: 記事本文の作成、保存、最終チェック、進捗更新まで（シリーズ/単発を問わない）。

---

## 2. 出力仕様（ファイル・命名・配置）
- 形式: Markdown（UTF-8, LF）。
- 保存先: シリーズ運用時は `{{SERIES_DIR}}`、単発の場合は適切なカテゴリ配下（例: HPブログ記事/症状別/）。
- ファイル名:
  - 構成案がある場合: 各回の固定タイトルと同一名（.md）。
  - 構成案がない場合: タイトル.md（確定後に必要ならリネーム）。
- 記事長: 目安3000文字。スマホ可読性重視で短段落。
- 医療安全: 誇大表現禁止。必要に応じて注意書き（一般情報である旨、個別診断ではない旨）を末尾に付与。
- 記事長: 目安3000文字。スマホ可読性重視で短段落。

---

## 3. 文体・トーン（参照1に完全準拠）
- 一人称: 「僕」。
- 敬体: です・ます調。要所で「〜なんです」「〜ですよね」など自然な語りかけ。
- 専門性×平易さ:
  - 専門用語は正確に使用（例: 中枢神経感作, 関節位置覚, ROM）。
  - 用語提示の直後に平易な説明・比喩を必ず補う。
    - 例: トリガーポイント＝“筋肉の中の小さな結び目”
    - 例: 筋膜癒着＝“洗濯物が絡まったようにくっつく”
- 視覚設計: 見出し（##/###）、太字、箇条書きで論点を整理。段落は短く。

備考: 「色付け（赤）」は媒体側仕様に依存。Markdown単体では過度な色指定は避け、太字・見出し・箇条書きを基本とする。

---

## 4. 構成パターン（シリーズ有無で使い分け）
シリーズ構成案（`{{STRUCTURE_FILE}}`）がある場合はその意図を優先。ない場合は以下の標準パターンを用いる。

1) 【Problem】問題提起（共感の導入）
- 読者の具体的な悩みや疑問を代弁し、共感を明確化。

2) 【Agitation】背景と放置リスクの解像度を上げる
- なぜ起こるのか（メカニズム）を研究・事例で説明。

3) 【Solution】解決の方向性・専門的視点
- 科学的根拠とともに、価値ある解決の道筋を提示。

4) 【Narrow down】今日からできる具体策
- 行動ハードルを下げたToDo（番号付き/箇条書き）。

5) 【Action】行動喚起と励まし
- まとめ＋「少しずつで大丈夫」など前向きな一言で締め。

---

## 5. 執筆テンプレート（コピペ可）
以下テンプレートに、シリーズ構成案の要点・引用資料（ある場合）を反映して埋める。

```
## はじめに（Problem）
読者の悩み・状況への共感から導入。「僕」一人称、です・ます調。

## なぜ起こるのか（Agitation）
- 用語：まず専門用語を正確に提示（例：中枢神経感作）。
- 直後に平易な説明（例：「神経の警報システムが過敏になっている状態」）。
- 研究や数値の要点を引用して信頼性を担保。

## 解決の方向性（Solution）
- 専門家視点での評価・介入の方針を簡潔に。
- 背景にある科学的根拠を短く添える。

## 今日からできること（Narrow down）
1. 〜を3回（目安5分）
2. 〜は避ける／〜を意識する
3. 呼吸・セルフケアなど、安全で再現性の高い内容

## まとめと次の一歩（Action）
- 要点の再掲。
- 「できることから少しずつ」など前向きなメッセージ。
```

## 6. コンテンツタイプ別テンプレ（用途で選択）
- 研究レビュー（Evidence解説）
```

## 何が言えるか（示唆）
## 限界と注意点（一般化の範囲）

```

- How To / セルフケア
```
## 注意事項（安全・禁忌）
## 前提（対象・目的）
## 手順（番号付きで具体的に）
## 頻度・回数・所要時間の目安
## よくある失敗と対策
```

- 症例ストーリー（匿名事例）
```
## 来院背景と主訴
## 評価（所見と仮説）
## 介入（意図と実施）
## 結果（変化と解釈）
## 学び（一般化できるポイント）
```

## 7. 作業フロー（参照2を統合・詳細化）
1) 参照読み込み（必須）
- 文体: `{{STYLE_GUIDE_FILE}}`
- 構成: `{{STRUCTURE_FILE}}`（シリーズで異なる）
- 運用: `{{OPS_GUIDE_FILE}}`（進捗も確認）

2) 対象回の確定
- 構成案の該当回を読み、固定タイトル・論点・引用資料を把握。
  - 代替: 構成案が未整備の場合は、PASONAの流れに沿って仮アウトラインを作成し、固定タイトルは仮置き→後で確定・リネーム。

3) 下書き作成（テンプレ準拠）
- PASONAの流れを守り、短段落・見出し・箇条書きで簡潔に。
- 専門用語→直後に平易化（必須）。

4) 根拠の明確化
- 「出典/参考」行で論文・レポート名を明記（既存ファイル名で可）。
  - 例: 出典: 「交通事故後のむち打ち症…包括的分析レポート」

5) 文体チェック（必須）
- 一人称「僕」、です・ます調、語りかけの自然さ。
- 段落の長さ、見出し階層、太字の要点強調。

6) 保存
- 保存先・命名は「2. 出力仕様」に従う。

7) 進捗更新（必須）
- OPSガイド（`{{OPS_GUIDE_FILE}}`）のチェックリストを更新。

## 9. 用語・表記の運用
- 英数字・略語: 初出で正式名→略称（例: 関節位置覚（JPS））。
- 数値: 「約」「目安」を添えて過度な断定を避ける。
- 禁止: 誇張表現、断定的な効果保証、他者誹謗。

## 11. 参考（よくある落とし穴）
- 専門用語の“説明忘れ”や“後回し”。→ 用語直後に一文で説明すること。
- 段落が長文化。→ 2〜4文で改行、視線誘導を意識。
- 構成案の引用資料が本文に反映されない。→ 4) 根拠の明確化を徹底。
- タイトルの微修正。→ 固定タイトルは変更不可。
