# PBLブログ記事執筆マスターガイド

このドキュメントは、PBL（フィジカルバランスラボ整体院）のブログ記事を、誰が書いても同じ品質で再現できるようにする統合ガイドです。

---

## 1. 文体・トーン

### 基本スタイル
- **一人称**: 「僕」
- **敬体**: です・ます調
- **トーン**: 専門性 × やさしさ × 励まし
- **語りかけ**: 「〜なんです」「〜ですよね」など自然な語りかけを適宜使用
- **ミニエピソード**: 匿名の短いエピソードを各記事で1つ以上

### 読みやすさ（モバイルファースト）
- **段落**: 2-4文で改行
- **見出し**: `##` と `###` を使用
- **太字**: 重要語の最小限強調
- **記事長**: 目安3000文字

### ナラティブ比率
- 本文の **70%以上** はプレーンな段落で書く
- 箇条書きは主に「今日からできること」セクションのみ
- 箇条書きは可能なら番号付き、1セクション最大5項目

### 専門用語の扱い
専門用語は正確に使用し、**直後に平易な説明・比喩を必ず補う**。

| 専門用語 | 平易な説明 |
|---------|----------|
| 神経筋制御 | 筋肉と神経のチームワーク |
| ウィークリンク | 動作の鎖の中の弱い環 |
| 免荷 | 体重の一部を預けて負担を軽くする工夫 |
| 不安定性 | 小さな揺れで微調整を引き出す刺激 |
| トリガーポイント | 筋肉の中の小さな結び目 |
| 筋膜癒着 | 洗濯物が絡まったようにくっつく |

## 2. 記事構成パターン（PASONA）

### 標準フロー
1. **【Problem】問題提起** - 読者の悩みを代弁し、共感を明確化
2. **【Agitation】背景説明** - なぜ起こるのか、放置リスクを穏やかに明示
3. **【Solution】解決の方向性** - 科学的根拠とともに価値ある道筋を提示
4. **【Narrow down】今日からできること** - 行動ハードルを下げた具体策3項目
5. **【Action】行動喚起** - 要点再掲と前向きな一言で締め

> **注意**: 見出しに（Problem）（Agitation）等のラベルは記載しない

### 各セクションの要件

#### Problem（はじめに）
- 読者の悩みの代弁、安心の提供
- 「僕」で語りかける
- 匿名の短いエピソードを1つ
- 最小2段落、箇条書き禁止

#### Agitation（なぜ起こるのか）
- 専門用語→直後に平易説明
- 比喩を1つ以上
- 放置リスクは脅かさず淡々と
- 最小2段落、箇条書き禁止

#### Solution（解決の方向性）
- 評価→介入→再評価の流れを文章で具体化
- 痛みゼロ原則の明記
- 段階的進行の説明
- 最小2段落、箇条書き禁止

#### Narrow down（今日からできること）
- 3項目を番号付きで
- 目安（回数/時間）を明記
- 痛みが出たら中止を記載
- 安全で再現性が高い内容

#### Action（まとめと次の一歩）
- 要点をやさしく再掲
- 「できることから少しずつ」など前向きなメッセージ
- 必要なら相談/予約への導線を自然に提示
- 最小1段落、箇条書き禁止

## 3. 記事テンプレート

### 3.1 シリーズ記事（PASONA＋ナラティブ）

```markdown
## はじめに
読者の悩みを代弁し、匿名の短いエピソードを1つ入れる。僕一人称、です・ます調。

## なぜ起こるのか
専門語→直後に平易説明。比喩を使い、放置リスクは脅かさず淡々と。

## 解決の方向性
評価→介入→再評価の流れを文章で具体化。痛みゼロ原則と段階進行。

## 今日からできること
1. 〜（目安：回数/時間）
2. 〜（痛みが出たら中止）
3. 〜（安全で再現性が高い）

## まとめと次の一歩
要点をやさしく再掲し、前向きな一言で締める。必要なら相談/予約への導線を自然に提示。
```

### 3.2 単発記事テンプレート

#### 研究レビュー
```markdown
## 背景
## 研究の要点（対象/手法/主要結果）
## 何が言えるか（示唆）
## 限界と注意点（一般化の範囲）
## 実践への翻訳（読者が取れる一歩）
```

#### How To / セルフケア
```markdown
## 注意事項（安全・禁忌）
## 前提（対象・目的）
## 手順（番号付きで具体的に）
## 頻度・回数・所要時間の目安
## よくある失敗と対策
```

#### 症例ストーリー（匿名事例）
```markdown
## 来院背景と主訴
## 評価（所見と仮説）
## 介入（意図と実施）
## 結果（変化と解釈）
## 学び（一般化できるポイント）
```

## 4. 作業フロー

### 4.1 シリーズ記事の場合

1. **参照ファイルの読み込み（必須）**
   - 本ガイド（文体・トーン）
   - シリーズ構成案ファイル（構成・内容の順序）

2. **対象回の確定**
   - 構成案の該当回を読み、固定タイトル・論点・引用資料を把握
   - 前後の記事内容とのつながりを意識

3. **下書き作成（テンプレ準拠）**
   - PASONAの流れを守り、短段落・見出し・箇条書きで簡潔に
   - 専門用語→直後に平易化（必須）

4. **根拠の明確化**
   - 「出典/参考」行で論文・レポート名を明記

5. **文体チェック（必須）**
   - 一人称「僕」、です・ます調、語りかけの自然さ
   - 段落の長さ、見出し階層、太字の要点強調

6. **保存**
   - ファイル名: `NN_固定タイトル.md`（NNは2桁連番）
   - 形式: Markdown（UTF-8, LF）

7. **進捗更新（必須）**
   - シリーズ管理表のチェックリストを更新

### 4.2 単発記事の場合

1. **テーマ確認**: 目的/読者像/前提条件を明確に
2. **テンプレ選択**: 研究レビュー/HowTo/症例から選択
3. **下書き作成**: PASONAの流れで約3000文字
4. **文体チェック**: 本ガイドに照らし合わせ確認
5. **保存**: 適切なカテゴリ配下に `タイトル.md` で保存

## 5. 品質チェックリスト（公開前必須）

- [ ] 一人称が「僕」で統一されている
- [ ] です・ます調で、語尾に不自然な揺れがない
- [ ] 専門用語→直後の平易説明・比喩が必ず付いている
- [ ] ナラティブ比率が0.7以上（箇条書きは主にNarrowDownのみ）
- [ ] 評価→介入→再評価の流れが文章で説明されている
- [ ] 見出し（##/###）と段落長がスマホ向けに最適化
- [ ] 「今日からできること」が3項目・安全・再現性高い
- [ ] 誇大表現の排除と安全ディスクレーマーの明記
- [ ] 見出しに（Problem/Agitation等）のラベル表記がない
- [ ] ファイル名・保存先が正しい

## 6. 安全・コンプライアンス

### 必須事項
- **ディスクレーマー**: 本文末に以下を記載
  > 本記事は一般情報であり、個別の診断・治療を提供するものではありません。痛みや違和感が出たら中止し、必要に応じて専門家へご相談ください。

- **痛みゼロ原則**: 施術・セルフケアで痛みを押し付けない
- **禁忌事項の記載**: 該当する場合は必ず注意喚起

### 禁止事項
- 誇大表現（「完治」「100%」「即効性」等）
- 断定的な効果保証
- 他者誹謗
- 数値の断定（「約」「目安」を添える）

## 7. シリーズ構成案テンプレート

新しいシリーズを企画する際は、以下のテンプレートを使用してLLMにシリーズ構成案を生成させます。

### 入力パラメータ
- `SERIES_NAME`: シリーズ名（例: むち打ち症シリーズ）
- `ARTICLE_COUNT`: 記事本数（例: 5 / 8 / 10）
- `AUDIENCE`: 読者像（年齢層/関心/悩み）
- `GOAL`: 読了後に実現したい変化
- `SCOPE_BOUNDARY`: 扱う範囲と除外範囲
- `KEY_TOPICS`: 必ず含めたい主要トピック
- `EVIDENCE`: 参考にしたい資料の種類や既存ファイル

### 生成プロンプト例

```
以下の入力パラメータで、PBLのスタイルガイドに準拠したシリーズ構成案を作ってください。

- SERIES_NAME: {{SERIES_NAME}}
- ARTICLE_COUNT: {{ARTICLE_COUNT}}
- AUDIENCE: {{AUDIENCE}}
- GOAL: {{GOAL}}
- SCOPE_BOUNDARY: {{SCOPE_BOUNDARY}}
- KEY_TOPICS: {{KEY_TOPICS}}
- TONE: 僕・ですます調・語りかけ
- EVIDENCE: {{EVIDENCE}}

要件:
- PASONAに沿い、各回に『今日からできること』を1〜3項目入れる。
- 専門用語には直後に平易説明を付ける。
- タイトルはそのままファイル名にできる形にする。
- スマホで読みやすい短文・短段落を想定する。
```

### 出力フォーマット

```
【シリーズ概要】
- シリーズ名: {{SERIES_NAME}}
- 対象読者: {{AUDIENCE}}
- ゴール: {{GOAL}}
- 本数: {{ARTICLE_COUNT}}
- 扱う範囲/除外範囲: {{SCOPE_BOUNDARY}}
- 主要トピック: {{KEY_TOPICS}}

【全体設計（ロードマップ）】
- 物語構造（導入→理解深化→応用/実践→総括）
- 各回の役割（認知/理解/行動/信頼形成 など）
- 期待行動（問い合わせ/継続読了/セルフケア着手 等）

【各回アウトライン】
第N回：＜固定タイトル案＞
- 目的（読者の変化）:
- 要点（3〜5項目・PASONA配置を意識）:
- 重要用語（平易説明付き）:
- 参考・出典（あればファイル名/文献名）:
- メモ（注意点/禁忌/次回への接続）:

…（記事数ぶん繰り返し）
```

## 9. よくある落とし穴

| 問題 | 対策 |
|-----|-----|
| 専門用語の説明忘れ・後回し | 用語直後に一文で説明 |
| 段落が長文化 | 2〜4文で改行、視線誘導を意識 |
| 構成案の引用資料が本文に反映されない | 根拠の明確化ステップを徹底 |
| タイトルの微修正 | 固定タイトルは変更不可 |
| 箇条書きの多用 | ナラティブ比率0.7以上を維持 |

## 10. 用語・表記の運用

- **英数字・略語**: 初出で正式名→略称（例: 関節位置覚（JPS））
- **数値**: 「約」「目安」を添えて過度な断定を避ける
- **ファイル形式**: Markdown（UTF-8, LF）
- **ファイル命名**:
  - シリーズ: `NN_固定タイトル.md`（NNは2桁連番）
  - 単発: `タイトル.md`
