# Part 1: Googleビジネスプロフィール（MEO対策）

## 1.1 概要

### 目的
1. **地域×症状×独自性** の関連性を継続強化
   - エリア: 名古屋市/千種区/名東区/星ヶ丘
   - 症状: 腰痛/五十肩/肩こり/脊柱側弯症
   - 独自性: レッドコード整体
2. **行動誘発**（経路/電話/予約）を最大化
3. **会話型検索/LLMO対応**（FAQ形式・自然文・一次情報）

### 成功指標（KPI）
- 主KPI: 経路リクエスト/電話タップ/Web予約遷移（週次）
- 副KPI: 指名検索・写真閲覧

---

## 1.2 投稿スケジュール（週3固定）

| 曜日 | 時間 | 狙い |
|-----|------|-----|
| 火 | 12:30 | 昼休み帯／回遊増 |
| 木 | 17:30 | 退勤前／予約意欲帯 |
| 土 | 14:30 | 週末来院ニーズ直撃 |

- 例外: 祝日/イベント週は日 10:00 追加可（週4）
- **SLA**: 口コミ/Q&A 返信は 48h以内

---

## 1.3 コンテンツ・ピラー（12カテゴリー）

1. 施術技術紹介（レッドコード/免荷/安全性）
2. 症例改善事例（匿名・数値・期間を明示）
3. 健康豆知識（姿勢/予防/セルフケア）
4. お客様の声（許諾/引用/写真配慮）
5. FAQ（所要時間/頻度/服装/駐車/学生対応）
6. 初回案内・アクセス（駅徒歩/駐車/所要時間）
7. 季節/イベント（冷え/花粉/マラソンなど）
8. 資格/学会/実績（専門性・信頼）
9. スタッフ/院内紹介（人となり/安心）
10. セルフチェック（30秒/壁面4点接触 等）
11. 地域連携（ランナー/周辺施設）
12. 予約状況/次月告知（空き枠/注意）

> 1ヶ月で12カテゴリーを一巡（偏り回避）

---

## 1.4 投稿の文字量・構成・トーン

### 基本ルール
- **文字量**: 本文 220–380字（スマホ一気読み）
- **構成**: ①結論（利益/独自性）→ ②根拠/一次情報 → ③FAQ 1問（会話文） → ④CTA
- **トーン**: 安心/共感/専門性。断定は避け「目安/個人差」で調整
- **文体**: 平易語優先（免荷=「体重を一部預ける」等の注釈）

### キーワード配置
- 会話型検索: 「どれくらい？」「何回？」等の自然文を1つ含める
- 地域×症状語: 冒頭50字内に1回、本文中に2–3回（不自然ゼロ）
- ハッシュタグ: 3–5個（例: #名古屋整体 #千種区 #名東区 #星ヶ丘 #腰痛）

## 1.5 タイトル生成パターン

| パターン | 形式 |
|---------|-----|
| P1 | 「{地域}の{症状}に——{独自性/利益}」 |
| P2 | 「{季節/状況}×{症状}：{30秒ケア/チェック}」 |
| P3 | 「{症状/対象}：{期間/回数の目安}で{体感/変化}」 |
| P4 | 「{施術名/機器}を整体に——{安全/再発予防}」 |

**例**:
- 「千種区の腰痛に——免荷で"痛みゼロ範囲"の再学習」
- 「会議前1分×肩こり：吐き切る→肩が下りる→胸を開く」
- 「名東区/五十肩：3週で夜間痛が軽くなった例（個人差）」

# Part 2: AI検索最適化（LLMO対策）

## 2.1 概要

### 目的
- 入力された日本語記事を「TL;DR 付き HTML」と「構造化 JSON-LD(@graph)」に自動変換
- HTML はスマホで読みやすいセマンティックタグ & class 構造
- 生成AI（ChatGPT/Claude等）とGoogle検索の両方で最適化

### LLMO/GEO（生成AI最適化）の観点
- エンティティ（about/mentions/alternateName）と手順（HowToStep）を機械可読に明示
- E-E-A-T（経験・専門性・権威性・信頼性）シグナルを Person/Organization スキーマで強化
- 会話型クエリ（質問形式の検索）に対応した見出し・構造
- AIクローラー（GPTBot/ClaudeBot/PerplexityBot）が正しく解析できるセマンティックHTML

## 2.4 生成ルール

### タイトル & TL;DR
- `{{title}}`: 最初の見出し
- `{{tldr}}`: 全角80–120字、1文、数字または具体的根拠を1つ以上含める
- `description`: TL;DR を短く自然に整えた120字前後

### 本文HTML
- 親ラッパ: `<main><article itemscope itemtype="https://schema.org/Article">…</article></main>`
- 目次（自動）: 最初の `<h2>` 群からアンカー付きの簡易ToC
- 見出し変換: `##`→`<h2 id="...">`、`###`→`<h3 id="...">`
- 段落: 空行2つで `<p>`
- 箇条書き: `- ` → `<ul><li>`、`1. ` → `<ol><li>`
- 医療記事の場合、本文末に「免責と受診の目安」を短文で追記

### 抽出ロジック

#### FAQ候補
以下にマッチした行を「質問」、次の非空段落を「回答」とする（最大5件）:
- `^\s*(Q[:：．。]|\[?Q\]?|質問[:：])` で始まる
- 行末が `？|?` で終わる
- 見出しに「よくある質問」「FAQ」が含まれる直下の項目

#### HowTo見出し
`やり方|手順|方法|セルフケア|ストレッチ|エクササイズ|トレーニング|対処|プロトコル`

#### 医療/健康キーワード
`腰痛|肩こり|頸|頚|頭痛|めまい|しびれ|血圧|炎症|関節|筋膜|MRI|レントゲン|脊柱|胸椎|頚椎|腰椎|むち打ち|WAD`

## 2.5 JSON-LD（@graph）

### 必須/条件付きノード

| ノード | 条件 |
|-------|-----|
| `WebPage` | 常に出力 |
| `Article` | **必須** |
| `FAQPage` | Q/A が1件以上抽出できた場合のみ |
| `HowTo` | 手順が抽出できた場合 |
| `MedicalCondition` | 健康キーワード検出時 |
| `Organization` | publisher として常に出力 |
| `BreadcrumbList` | CONTEXTにあれば |
| `ImageObject` | 代表画像があれば |

### E-E-A-T強化用 Person スキーマ
- `sameAs`: 公式プロフィールURL
- `knowsAbout`: 専門分野の配列
- `alumniOf`: 出身校
- `hasCredential`: 資格情報
- `worksFor`: 所属組織

### 出力形式
- 厳密なJSON、1行に最小化
- `@id` は `url#article` など URL ベースのフラグメント

---

## 2.6 会話型クエリ対応（LLMO/GEO最適化）

### 見出しの最適化
- **見出しは疑問形を優先**: 「〜とは？」「どうすれば〜？」「なぜ〜？」

### 本文の最適化
- **冒頭で直接回答**: 各セクションの最初の1-2文で質問に直接回答
- **段落は80-120文字**: AI抽出に最適な長さ
- **情報密度の向上**: 統計・数値・引用・具体例を積極的に含める

### エンティティの一貫性
- 同一概念は同じ表記で統一
- 初出時のみ `alternateName` で別名を併記

---

## 2.7 AIクローラー対応の注意

- **サーバーサイド出力必須**: JSON-LDは`<head>`内にサーバーサイドで直接出力
- **セマンティックHTML**: `<article>`,`<section>`,`<nav>`,`<header>`,`<footer>`等を適切に使用
- **見出し階層**: H1→H2→H3の階層を厳守、スキップしない
